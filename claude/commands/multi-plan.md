# Multi-Plan

對同一需求從三個工程角色並行分析，彙整各角度優缺點後輸出最優推薦方案。

## 三角色並行分析

使用 Agent tool 同時啟動以下三個子分析（不串行等待）：

**角色 A — 架構師視角**
關注：可擴展性、模組邊界、技術債、長期維護成本。
輸出：方案草圖 + 模組依賴圖 + 潛在架構債。

**角色 B — 實作工程師視角**
關注：實作複雜度、現有代碼複用、交付速度、測試難度。
輸出：最快可行的實作路徑 + 受影響檔案清單。

**角色 C — 維護工程師視角**
關注：可讀性、可觀測性、除錯難度、文件需求、Oncall 風險。
輸出：維護成本評估 + 監控 / 日誌需求 + 風險清單。

## 彙整原則

收到三份分析後：
1. 找出三角色都認同的方向（高信度共識）
2. 標記分歧點，說明取捨理由
3. 若有明顯優勢方案則直接推薦，否則提供 2 個選項供用戶決策

## 輸出格式

```
MULTI-PLAN: <需求標題>
════════════════════════════════════════

[架構師] <方案摘要，2-3 句>
優點：<列點>  缺點：<列點>

[實作工程師] <方案摘要，2-3 句>
優點：<列點>  缺點：<列點>

[維護工程師] <方案摘要，2-3 句>
優點：<列點>  缺點：<列點>

────────────────────────────────────────
共識：<三角色共同認為重要的點>

分歧：<角色 A vs B vs C 的主要分歧>

RECOMMENDATION: <方案名稱>
理由：<為什麼這個方案在三個維度取得最佳平衡>
取捨：<採用此方案放棄了什麼>
════════════════════════════════════════
```

用戶確認推薦方案後，可直接呼叫 `/plan` 生成詳細執行步驟。
