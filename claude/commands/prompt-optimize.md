# Prompt Optimize

對用戶提供的 prompt（或當前 command / agent 檔案）進行品質診斷與改寫。

`$ARGUMENTS` 可傳入目標 prompt 文字，或省略以分析當前對話最近一條指令。

## 五維評分

| 維度 | 說明 | 滿分 |
|------|------|------|
| 清晰度 | 意圖明確，無歧義 | 20 |
| 具體性 | 有具體參數、範圍、格式要求 | 20 |
| 約束 | 明確說明不要做什麼 | 20 |
| 範例 | 有 input/output 範例或 few-shot | 20 |
| 格式 | 指定輸出結構（JSON / Markdown / 純文字）| 20 |

## 常見問題診斷

- **模糊動詞**：「幫我處理」→「提取 / 轉換 / 驗證 / 過濾」
- **缺少約束**：沒說不要做什麼，AI 會自由發揮填充
- **過長背景**：超過 200 字的背景描述通常可壓縮至 50 字以內
- **無輸出格式**：未指定格式時 AI 會選擇最長的回應方式
- **混合多個任務**：一個 prompt 超過一個核心意圖應拆分

## 輸出格式

```
PROMPT QUALITY ANALYSIS
────────────────────────────────────────
清晰度    XX/20   <一行診斷>
具體性    XX/20   <一行診斷>
約束      XX/20   <一行診斷>
範例      XX/20   <一行診斷>
格式      XX/20   <一行診斷>
────────────────────────────────────────
總分      XX/100  <GOOD ≥80 / FAIR 60-79 / POOR <60>

主要問題：
1. <問題描述>

優化後版本：
---
<重寫後的 prompt>
---

節省估算：~XX% tokens，預期回應品質提升 <描述>
```

改寫時保持原始意圖不變，只改善表達方式與結構。
