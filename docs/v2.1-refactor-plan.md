## 背景資料

### Claude Code 源碼洞察（來自 AlvinBian/claude-code-source-code）

1. **Rules 加載機制**：所有 ~/.claude/rules/*.md 在每次對話開始時注入到 system prompt，直接增加 input token
2. **CLAUDE.md 加載**：fetchSystemPromptParts() 會讀取 CLAUDE.md 作為 memory context 注入
3. **.claudeignore**：應用於 glob/grep 操作的文件過濾，排除的目錄不會被 Claude 掃描
4. **Auto-Compact**：context 接近上限時自動壓縮歷史訊息
5. **Hooks 執行順序**：PreToolUse hooks → permission rules → 互動確認 → 工具執行
6. **settings.json env**：env 區塊的變數會注入到 Claude session 的環境中
7. **Permission 規則**：alwaysAllowRules / alwaysDenyRules / alwaysAskRules 三層

**問題**：14 個 rules 全部自動加載到每次對話的 system prompt

**當前 14 個 rules：**

- slack-mrkdwn（Slack 專用）

**預期收益**：14 → 6 個 rules，減少 ~60% rules input token

**問題**：當前 generate-claude-md.mjs 生成的內容可能過長

## 補充資料（2026-04-02 更新）

### 上下文壓縮算法（來源：win4r/cc-notebook）

Claude Code 使用四層壓縮架構：
1. **Microcompact**（規則清理，<1ms）：保留最近 N 個工具結果，舊的替換為 `[Old tool result content cleared]`
2. **Auto-Compact**（閾值觸發）：token 接近上限時自動壓縮，Opus 200K 在 ~167K 觸發
3. **Full Compact**（Fork Agent 生成結構化摘要，5-30s）
4. **Session Memory**（重用已有摘要，<10ms）

**壓縮時保留**：用戶意圖、技術概念、文件名、代碼片段、未完成任務
**壓縮時丟棄**：`<analysis>` 區塊、圖片/文檔（替換為 token）、重注入附件

**對 ab-dotfiles 的啟示**：
- Rules 每次都注入 system prompt，越長越費 token → 必須精簡
- CLAUDE.md 是 memory context，也會被注入 → 控制長度
- 設定 `CLAUDE_AUTOCOMPACT_PCT_OVERRIDE` 可以調整觸發閾值

### ccleaks.com 完整分析

**關鍵發現（可應用到 ab-dotfiles）：**
- **32 個 feature flags**：REACTIVE_COMPACT / CONTEXT_COLLAPSE / HISTORY_SNIP / CACHED_MICROCOMPACT / TOKEN_BUDGET 都是壓縮相關
- **AUTOCOMPACT_PCT_OVERRIDE**：可設定在 settings.json env 中覆蓋壓縮閾值
- **MAX_THINKING_TOKENS**：可限制思考 token（settings-template 已設定 31999）
- **Plan Mode V2**：3 個並行 agent（Max/Team），CLAUDE_CODE_PLAN_V2_AGENT_COUNT 可覆蓋
- **Permission 系統四層**：default → auto（YOLO classifier）→ bypass → yolo
- **26 個隱藏命令**：/ctx-viz（context 可視化）、/force-snip（強制壓縮）、/dream（記憶整合）
- **Secret CLI Flags**：--bare（無 hooks/plugins/memory）、--dump-system-prompt（打印系統提示）

### 核心策略：複用 > 新寫

**原則**：最大程度複用 Claude Code 內建的 skills/commands/rules/agents，只在流程編排和決策層優化。

**Claude Code 內建能力（不需要自己寫）：**

- 53 個內建工具（Bash、FileRead、FileEdit、Glob、Grep、Agent 等）
- 內建壓縮系統（四層自動壓縮）
- 內建權限系統（四層門控）
- 內建 memory 系統（auto-memory）

**ab-dotfiles 應該做的（流程編排層）：**
1. **Rules**：精簡為 Claude 不知道的「項目特定規範」，不要重複 Claude 已知的通用規範

5. **settings.json**：精準設定 permissions + env（壓縮閾值、思考上限）
6. **.claudeignore**：每個 repo 自動生成，大幅減少 token 消耗
