# Aside

Bounded exploration — 在不修改任何檔案、不推進主任務的前提下，回答一個快速問題或驗證一個假設。

`$ARGUMENTS` 傳入要探索的問題或假設。

## 原則

- **唯讀**：aside 期間不寫入、不修改任何檔案
- **有界**：只用 ≤ 3 個工具呼叫完成探索，避免 context 膨脹
- **明確結束**：探索完成後明確宣告「回到主任務」
- **不改變狀態**：aside 的發現不自動觸發後續行動，除非用戶明確要求

## 適用情境

| 情境 | 範例 |
|------|------|
| 驗證假設 | 「這個函式有沒有被其他地方呼叫？」 |
| 快速查詢 | 「這個套件的 API 是什麼？」 |
| 影響評估 | 「改這個 type 會影響哪些檔案？」 |
| 概念確認 | 「這個模式叫什麼名字？」 |

## 執行流程

1. 宣告：`[ASIDE] 探索：<問題>`
2. 執行唯讀工具（Grep / Read / Glob，至多 3 次）
3. 回答問題，1-3 句話結論
4. 宣告：`[END ASIDE] 回到主任務：<主任務描述>`

## 輸出格式

```
[ASIDE] 探索：<問題>
──────────────────────────
發現：<1-3 句話結論>
影響主任務：<有 / 無，如有說明如何影響>
──────────────────────────
[END ASIDE] 回到主任務：<繼續主任務>
```

aside 的結論只在需要時才整合進主任務流程，預設不改變原有計畫。
