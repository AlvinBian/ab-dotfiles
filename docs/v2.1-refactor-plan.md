## 背景資料

### Claude Code 源碼洞察（來自 AlvinBian/claude-code-source-code）

1. **Rules 加載機制**：所有 ~/.claude/rules/*.md 在每次對話開始時注入到 system prompt，直接增加 input token
2. **CLAUDE.md 加載**：fetchSystemPromptParts() 會讀取 CLAUDE.md 作為 memory context 注入
3. **.claudeignore**：應用於 glob/grep 操作的文件過濾，排除的目錄不會被 Claude 掃描
4. **Auto-Compact**：context 接近上限時自動壓縮歷史訊息
5. **Hooks 執行順序**：PreToolUse hooks → permission rules → 互動確認 → 工具執行
6. **settings.json env**：env 區塊的變數會注入到 Claude session 的環境中
7. **Permission 規則**：alwaysAllowRules / alwaysDenyRules / alwaysAskRules 三層

## 階段 1：Token 優化（P0 — 影響最大）

### 1.1 Rules 合併精簡
**問題**：14 個 rules 全部自動加載到每次對話的 system prompt
**目標**：合併為 5-6 個核心 rules，每個更精煉

**當前 14 個 rules：**
- code-style, git-workflow, project-conventions, testing, performance（核心 5）
- slack-mrkdwn（Slack 專用）
- agents, hooks, patterns（元配置 3）
- api-conventions, error-handling, security-baseline, tool-selection（擴充 4）

**合併方案：**
1. `core-conventions.md` ← code-style + project-conventions + patterns
2. `workflow.md` ← git-workflow + testing + performance
3. `security.md` ← security-baseline + error-handling
4. `tooling.md` ← tool-selection + agents + hooks（決策樹 + 觸發詞）
5. `api.md` ← api-conventions（如果不做 API 開發可以不裝）
6. `slack-mrkdwn.md` ← 保持（Slack 專用，不用時可 disable）

**預期收益**：14 → 6 個 rules，減少 ~60% rules input token

### 1.2 CLAUDE.md 生成策略優化
**問題**：當前 generate-claude-md.mjs 生成的內容可能過長
**優化**：
- 控制總長度在 500 行以內
- 只包含：技術棧摘要、目錄結構（2 層）、常用命令、開發規範
- 移除冗長的依賴列表和 API 說明
- 使用分節結構，Claude 可按需讀取

### 1.3 新增 .claudeignore 模板
**問題**：缺少 .claudeignore，Claude 會掃描 node_modules/dist 等
**方案**：setup 時為每個 repo 生成 .claudeignore
```
node_modules/
dist/
build/
.next/
.nuxt/
coverage/
*.min.js
*.map
*.lock
pnpm-lock.yaml
package-lock.json
```

### 1.4 settings-template.json 優化
- 確保 CLAUDE_AUTOCOMPACT_PCT_OVERRIDE 設定
- model/effort 策略根據任務類型自動調整
