# Plan

理解需求 → 識別受影響範圍 → 拆解子任務 → 標記依賴 → 提示風險。

純規劃模式：**不修改任何檔案**，只輸出計畫供用戶確認。

## 執行步驟

1. **理解需求** — 用一句話重述目標，確認理解正確
2. **識別受影響檔案** — 用 Grep / Glob 快速掃描（< 5 個工具呼叫）
3. **拆解子任務** — 每個子任務單一職責，可獨立執行或測試
4. **標記依賴** — 標出哪些任務必須串行（A → B），哪些可平行
5. **風險提示** — 識別 breaking change、外部依賴、資料遷移等風險點

## 複雜度標準

| 標記 | 說明 |
|------|------|
| S | < 30 分鐘，單檔案改動 |
| M | 30 分鐘 ~ 2 小時，2-5 個檔案 |
| L | > 2 小時，跨模組或需要設計決策 |

## 輸出格式

```
PLAN: <任務標題>
────────────────────────────────────────
目標：<一句話說明>
受影響範圍：<N 個檔案 / N 個模組>

子任務：
1. [S] <任務描述>
   檔案：<path/to/file.ts>
   依賴：無

2. [M] <任務描述>
   檔案：<path/to/service.ts>, <path/to/types.ts>
   依賴：任務 1 完成後

3. [S] <任務描述>
   檔案：<path/to/test.ts>
   依賴：任務 2 完成後

執行順序：1 → 2 → 3（可平行：無）

風險：
- ⚠️ <風險描述，影響範圍，建議處理方式>

預估總工時：<S/M/L 加總>
────────────────────────────────────────
確認後輸入「開始」執行，或指定子任務編號單獨執行。
```
